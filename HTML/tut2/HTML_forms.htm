application/x-www-form-urlencoded (Default)

Description: This is the default form encoding. It encodes the form data into key-value pairs, where each key and value are URL-encoded (spaces become +, special characters are encoded in a URL-safe way).

Usage: Suitable for sending simple text and numeric data.

multipart/form-data

Description: This encoding type is used when the form includes file uploads. It sends the data in separate parts for each form field.

Usage: Required when your form includes file input (input type="file">).

text/plain

Description: Sends form data as plain text, without any encoding. This is rarely used because the data isn’t structured and not URL-encoded.

Usage: Generally used for debugging purposes or for applications that require raw text.
